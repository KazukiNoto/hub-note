# DrawIOワイヤーフレームを使用したAI駆動UI開発

**Source**: <https://qiita.com/R61/items/d7e1b9c5595265163861>

## 概要

テキストだけでUIの意図を伝えることは困難で、AIとの会話では誤解が繰り返される。DrawIOでワイヤーフレームを作成し「共通言語」として機能させることで、この課題を解決し、AI駆動UI開発を劇的にスムーズにできる手法について紹介しています。

## 課題

- テキスト要件定義とAI出力の間に認識のズレが発生
- レイアウトの曖昧さが伝わりにくい
- 要素の関係性が曖昧になりやすい
- UI状態の表現が困難
- 修正指示が抽象的になり、修正もきっかけになりやすい

## 解決策

**DrawIOワイヤーフレームを活用**

- XMLベースのDrawIO形式により、AIが「読める」視覚的設計ドキュメントを作成
- テキストの要件定義と視覚的な設計を分離
- ワイヤーフレームを統一的情報源（SSOT）として機能させる

## 効果

- 認識のズレが大幅に減少
- 修正指示がより具体的・明確になる
- イテレーション速度が向上
- 全画面の整合性レビューが容易化
- AIとの協働効率が向上

## 実践例

SmartAM（AI統合メールクライアント）プロジェクトで24ページのワイヤーフレームを活用して開発を実施し、成果を検証。

## まとめ

視覚的な設計（ワイヤーフレーム）を明示的に準備することで、AIとの開発では「共通言語」が形成され、結果として開発速度と品質の両面で大きな改善が期待できます。
