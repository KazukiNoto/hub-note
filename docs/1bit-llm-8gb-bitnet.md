# 1.58ビットに進化したから8GBで十分――BitNet系LLMの最前線

**Source**: <https://www.techno-edge.net/article/2026/04/17/4997.html>

## 概要

Microsoftが開発する **BitNet b1.58** に代表される「1.58ビットLLM」技術が、ローカルAIの常識を塗り替えつつある。重みを `{-1, 0, +1}` の三値で表現することで、従来の32ビット浮動小数点モデルに比べてメモリ使用量を劇的に削減し、**8GBのRAMで十分に動作する**時代が到来しようとしている。

## 1.58ビットとは何か

| 比較項目 | 従来のFP32モデル | BitNet b1.58 |
|---|---|---|
| 重みの表現 | 32ビット浮動小数点 | `-1 / 0 / +1` の三値 |
| ビット数の根拠 | — | log₂(3) ≈ **1.58ビット** |
| 計算方式 | 乗算＋加算 | **加算のみ**（乗算不要） |
| GPU依存 | 高い | CPUのみでも動作可能 |

三値量子化は学習後に圧縮する通常の量子化と異なり、**学習段階から三値で最適化**する「ネイティブ量子化」であるため、精度劣化が少ない。

## 主なポイント

- **メモリ使用量が10分の1以下**：BitNet b1.58 2B4T（20億パラメータ）は約0.4GBで動作。通常モデルの2〜5GBと比較して圧倒的に軽量
- **消費電力を最大93%削減**：加算のみの演算により電力効率が大幅に向上
- **8億パラメータがiPhoneで動作**：「Bonsai 8B」など1.58ビット系モデルは8BパラメータをわずかMacBook（8GB）で高速推論
- **CPU推論が現実的に**：GPU不要でサーバーレスや組み込み機器への展開が現実的になる
- **4兆トークンで学習**：BitNet b1.58 2B4Tは大規模データセットで学習済みで、同規模のフル精度モデルに匹敵する性能を発揮

## 技術的背景

BitNetアーキテクチャは **Microsoft Research** が主導し、2024年2月の論文 *"The Era of 1-bit LLMs"* で注目を集めた。その後、2025年に公開された **BitNet b1.58 2B4T** がオープンソースとして公開され、実用化が加速。

```
パラメータ数: 2B（20億）
学習トークン数: 4兆
メモリ使用量: 約0.4GB
対応デバイス: CPU（x86 / ARM）
```

!!! info "「1.58ビット」の由来"
    三値 {-1, 0, 1} を表現するのに必要な情報量は log₂(3) ≈ 1.585ビット。
    これを丸めて「1.58ビット」と呼んでいる。

## まとめ

1.58ビットLLMは単なる圧縮技術ではなく、**学習アーキテクチャ自体を再設計**するアプローチだ。8GBのRAMでも実用的なLLMが動く世界は、クラウド依存を減らし、プライバシーに配慮したエッジAIの普及を後押しする。Microsoftのオープンソース公開が起点となり、今後も軽量高性能モデルの競争が加速することが予想される。
