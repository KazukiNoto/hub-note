# 完全ローカル・モバイル完結型 リフレクティブ・メモリーAIアプリ

プライバシー、低コスト、パーソナライズを高い次元で両立する、Next.js製のオールオンデバイスAIアプリのコンセプトと実装戦略。

---

## 1. 基本アーキテクチャ

サーバーサイドの処理を一切持たない**静的サイト（SSG / `output: 'export'`）**として構築する。GitHub Pages等でホストでき、データはユーザーのデバイス内だけで循環する。

| レイヤー | 技術 |
|---|---|
| フロントエンド | Next.js (React) + Tailwind CSS |
| 推論エンジン | WebLLM (WebGPU) または Transformers.js (Wasm/WebGPU) |
| ストレージ | IndexedDB (Dexie.js) |
| バックグラウンド処理 | Web Worker（推論・内省処理のオフロード） |

---

## 2. リフレクティブ・メモリーの設計

モバイル端末の限られた計算資源（メモリ・バッテリー）を考慮し、IndexedDBを活用した**内省的記憶**を実装する。

### 短期記憶

直近の会話ログをIndexedDBから即座に読み込み、推論コンテキストを維持する。

### 長期記憶

過去の重要な事実やユーザーの嗜好を「タグ」や「要約」として永続保存する。

### リフレクション（内省）

ユーザーのアイドル時間にLLMがバックグラウンドで会話を振り返り、新たな知見を抽出してDBのユーザープロファイルを自動更新する。

```
会話ログ (短期)
      ↓ アイドル時に内省
ユーザープロファイル (長期) ← タグ・要約・嗜好
      ↓ 次回会話時に注入
推論コンテキスト
```

---

## 3. 実装のキーポイント

### モデル選定

モバイルでの安定稼働を目指し、**1B〜3Bパラメータ**のSLMに4-bit量子化を適用する。

!!! example "推奨モデル例"
    - **Qwen-1.5B** — 軽量・多言語対応
    - **Phi-3-mini (3.8B)** — 推論品質が高い
    - **Gemma-2B** — Google製・安定動作

数GBのRAM消費に抑えつつ、十分な推論性能を確保できる。

### UXの向上

- **PWA化** — オフライン動作とホーム画面追加でネイティブアプリに近い体験
- **ストリーミング表示** — トークン逐次出力でモバイルGPUの待ち時間を視覚的に軽減
- **キャッシュ管理** — 初回モデルダウンロード（約1〜2GB）の進捗バー表示とキャッシュ永続化

### Web Worker 構成

```
メインスレッド (UI)
    │
    ├─ IndexedDB (Dexie.js)  ← 会話・プロファイル保存
    │
    └─ Web Worker
           ├─ WebLLM / Transformers.js  ← 推論
           └─ リフレクション処理  ← アイドル時に起動
```

---

## 4. プロジェクトのメリット

=== "ゼロ・ランニングコスト"

    サーバーレス構成のため、ユーザー数が増えても運営コストが増えない。
    GitHub Pages等の無料ホスティングで完全運用可能。

=== "究極のプライバシー"

    会話データが端末の外に出ることがない。
    デリケートな思考の整理やメモに適している。

=== "自分専用の成長"

    使うほどにIndexedDBへ「ユーザーに関する知識」が蓄積され、
    LLMがより的確なアドバイスを行うようになる。
